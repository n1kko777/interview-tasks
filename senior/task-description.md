# Практическое задание: Рефакторинг + новая фича

## Контекст

Вам дан компонент `UserListPage` — экран со списком пользователей в Ionic React приложении. Код работает, но написан плохо: всё в одном файле, нет типов, неоптимальные паттерны.

Ваша задача — отрефакторить код и добавить новую функциональность.

---

## Часть 1: Рефакторинг (20 мин)

### 1.1 TypeScript-типизация
- Добавьте интерфейсы/типы для данных (`User`, props компонентов)
- Уберите все `any`

### 1.2 Декомпозиция на компоненты
Разбейте монолитный компонент на:
- `UserCard` — карточка пользователя
- `SearchBar` — строка поиска
- `UserDetailModal` — модалка с деталями

### 1.3 Кастомные хуки
Вынесите логику в хуки:
- `useUsers()` — загрузка и хранение списка пользователей
- `useFavorites()` — управление избранным

### 1.4 Ionic best practices
- Замените `useEffect` на `useIonViewWillEnter` для загрузки данных (в Ionic страницы кешируются и `useEffect` с `[]` сработает только один раз)

---

## Часть 2: Новая фича (15 мин)

### Debounced-поиск
Сейчас фильтрация происходит мгновенно при каждом нажатии клавиши. Реализуйте debounced-поиск:

1. Создайте кастомный хук `useDebounce(value, delay)`
2. Используйте задержку **300ms** между вводом и фильтрацией
3. Хук должен корректно очищать таймер (cleanup)

---

## Требования

- Весь код можно писать в одном файле (разделение на файлы не обязательно)
- Сохраняйте существующую функциональность (список, поиск, избранное, модалка)
- Используйте Ionic-компоненты (`IonCard`, `IonSearchbar`, `IonModal` и т.д.)
- Комментируйте свои решения, если считаете нужным

- Правильное использование React-паттернов (hooks, memo)
- Понимание Ionic-специфики
- Корректность реализации debounce
- Чистота и читаемость кода
